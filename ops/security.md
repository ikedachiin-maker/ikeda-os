# ops/security.md — セキュリティ設計（3本柱その1）

> 動画でつかさ氏が最も時間をかけた部分。「危ないから使わない」ではなく「対策して使う」。

## 基本方針

Mac mini + Tailscale構成のため、公開VPSを晒さない＝攻撃面が小さい。

## チェックリスト

- [ ] 認証必須化（橋の入り口に認証）
- [ ] APIキー・トークンは暗号化（環境変数 or キーチェーン。リポジトリにコミットしない）
- [ ] Tailscaleで内部ネットワークに閉じる（外部から直接叩けない）
- [ ] 不正アクセスの監視ログ → 異常があれば即 Telegram通知
- [ ] 定期アップデート
- [ ] `.gitignore` で `.env` / `*.key` / `secrets/` を除外（設定済み）

## 許容範囲のバランス

セキュリティを固めすぎると動かなくなる。最低限自分が許せる範囲を決める。
判断に迷うワードは「高校生でも分かる言葉」でTelegramに説明させ、自分で検索して納得してから採用する。
